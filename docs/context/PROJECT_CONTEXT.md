# AresForge Project Context

## Project Name

AresForge

## Vision

AresForge is a local-first AI software factory that can take ideas from intake to release with minimal manual effort.

It should mimic a full development team using specialized AI agents for intake, product analysis, architecture, planning, frontend, backend, DevOps, automation, QA, testing, documentation, monitoring, PR scoring, release, and escalation.

## Core Principle

AresForge must become its own first customer.

The project will bootstrap itself by using GitHub issues, documentation, validation workflows, local AI review, and eventually its own dashboard to manage future development work.

## Product Identity

AresForge is not a clone of ProjectForge or homelab-ai.

It is a new standalone product that can leverage both as references:

- ProjectForge provides governance, lifecycle, agent standards, issue standards, PR standards, and project management patterns.
- homelab-ai provides the Ares local runtime, Ollama, Open WebUI, GitHub runner, and local automation workflow patterns.
- Hermes Agent is a reference for agent architecture patterns.

## First Managed Project

The first managed project inside AresForge is AresForge itself.

Until the dashboard exists, GitHub and documentation are the temporary source of truth.
